## 1. Feature Overview

AI-powered personalized tutor for students that provides adaptive learning, homework help, concept explanation, and practice questions based on the student's curriculum, performance, and learning pace. Accessible via parent app (for younger students) or future student app.

### Goals

- Adaptive learning paths based on student's performance data
- Homework help: student photographs homework → AI explains concept (not just gives answer)
- Practice questions generated by AI, calibrated to student's level
- Concept explanation in student's preferred language
- Progress tracking with mastery levels per topic
- Teacher visibility into AI tutor usage and gaps identified

## 3. Functional Requirements

| ID | Requirement |
|---|---|
| FR-1 | Adaptive learning path: AI recommends next topics based on mastery + syllabus progress |
| FR-2 | Homework help: student asks question (text or photo) → AI explains concept step-by-step (no direct answers) |
| FR-3 | Practice questions: AI generates questions calibrated to student's level (easy/medium/hard) |
| FR-4 | Concept explanation: AI explains topic in student's language with examples |
| FR-5 | Mastery tracking: per-topic mastery level (0-100%) based on practice performance |
| FR-6 | Teacher dashboard: see which topics students are struggling with (aggregated) |
| FR-7 | Conversation history: student can revisit past tutoring sessions |
| FR-8 | Safety: AI refuses to solve homework directly, guides instead |
| FR-9 | Rate limiting: 50 questions per day per student (configurable) |

## 4. Database Design

```sql
CREATE TABLE ai_tutor_sessions (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    school_id       UUID NOT NULL,
    student_id      UUID NOT NULL,
    session_type    VARCHAR(16) NOT NULL,          -- homework_help | concept_explanation | practice | adaptive_path
    subject         TEXT NOT NULL,
    topic           TEXT,
    messages        TEXT NOT NULL,                 -- JSON: [{"role": "user/assistant", "content": "...", "timestamp": "..."}]
    model_used      VARCHAR(64),
    tokens_used     INTEGER,
    created_at      TIMESTAMP NOT NULL DEFAULT now()
);
CREATE INDEX idx_tutor_sessions_student ON ai_tutor_sessions(student_id, created_at DESC);

CREATE TABLE ai_tutor_mastery (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    school_id       UUID NOT NULL,
    student_id      UUID NOT NULL,
    subject         TEXT NOT NULL,
    topic           TEXT NOT NULL,
    mastery_level   REAL NOT NULL DEFAULT 0,       -- 0.0 to 1.0
    questions_attempted INTEGER NOT NULL DEFAULT 0,
    questions_correct INTEGER NOT NULL DEFAULT 0,
    last_practiced_at TIMESTAMP,
    created_at      TIMESTAMP NOT NULL DEFAULT now(),
    updated_at      TIMESTAMP NOT NULL DEFAULT now(),
    UNIQUE(student_id, subject, topic)
);
CREATE INDEX idx_tutor_mastery_student ON ai_tutor_mastery(student_id, subject);

CREATE TABLE ai_tutor_daily_usage (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id      UUID NOT NULL,
    date            DATE NOT NULL,
    question_count  INTEGER NOT NULL DEFAULT 0,
    tokens_used     INTEGER NOT NULL DEFAULT 0,
    UNIQUE(student_id, date)
);
```

### 5.2 System Prompts

**Homework help:**
```
System: You are VidyaSetu, an AI tutor for Indian school students.
Your role is to GUIDE, not to SOLVE. When a student asks a homework question:
1. Identify the concept being tested
2. Explain the concept with a simple example
3. Give a hint for the specific problem
4. Ask the student to try solving it
5. If they're still stuck, break it down step-by-step but don't give the final answer
Respond in {{language}}. Be encouraging and patient.

Context: Student is in {{grade}}, studying {{subject}}. Current topic: {{topic}}.
Mastery level for this topic: {{mastery}}%.
```

**Practice question generation:**
```
System: Generate {{count}} practice questions for a {{grade}} student in {{subject}}, topic: {{topic}}.
Difficulty: {{difficulty}} (easy=direct application, medium=multi-step, hard=complex problem).
Each question should have: question_text, options (if MCQ), correct_answer, explanation.
Output JSON array.
```

## 7. Safety & Content Guidelines

- AI must NOT solve homework directly — must guide and explain
- AI must NOT provide answers to exam questions
- AI must use age-appropriate language
- AI must stay within curriculum scope
- AI must refuse inappropriate requests
- All conversations logged for audit (DPDP compliance)
- Teacher can review AI tutor conversations for quality
